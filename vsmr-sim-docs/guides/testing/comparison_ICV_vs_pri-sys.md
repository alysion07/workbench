# MARS 입력 파일 비교 보고서
## 100%_ICV.i vs 100%_pri-sys.i

**비교일**: 2026-03-19
**원본 파일**: `100%_ICV.i` (수동 작성 원본, 15,316 lines)
**생성 파일**: `100%_pri-sys.i` (MARS GUI Editor v0.1.0 생성, 12,545 lines)
**참조 매뉴얼**: MARS-KS Code Manual, Volume II - Input Requirements (KINS/RR-1822, Vol.2, Rev.2, 2022.12)

## 2. 기능 차이 없는 형식적 차이

### 2.1 숫자 표기법 차이 (약 1,500건 이상)

GUI Editor가 과학적 표기법을 풀어서 출력.

| 카드 | ICV (원본) | PRI (GUI 생성) | 비고 |
|------|-----------|---------------|------|
| 1000200 | `15.125e6` | `15125000.0` | 동일 값 |
| 11200001 | `8.71e6` | `8710000.0` | 동일 값 |
| 401 W7 | `1.0e6` | `1000000.0` | 동일 값 |
| 410 W7 | `11.30e6` | `11300000.0` | 동일 값 |

**매뉴얼 Section 1.5**: 실수 필드는 `12.45`, `1.245E1`, `1.245e+1` 등 모두 동일하게 처리됨.
**판정**: 기능 차이 없음.

### 2.2 후행 0 및 소수점 차이 (약 300건)

| 카드 | ICV | PRI | 비고 |
|------|-----|-----|------|
| 1101101 W3 | `0.6450` | `0.645` | 동일 값 |
| 1103102 W1 | `0.060` | `0.06` | 동일 값 |
| 11001501 W6 | `29.20` | `29.2` | 동일 값 |

**판정**: 기능 차이 없음.

### 2.3 주석(Comment) 차이 (약 660건)

GUI Editor가 데이터 카드의 인라인 주석(`*lhj`, `*Average`, `* SMART330 : ...` 등)을 제거.

**매뉴얼 Section 1.4**: 주석은 `*` 또는 `$` 뒤의 내용으로, 입력 처리에 영향 없음.
**판정**: 기능 차이 없음.

### 2.4 카드 번호 포맷 차이

| 카드 | ICV | PRI | 비고 |
|------|-----|-----|------|
| Card 001 | `1  90` | `001  90` | 동일 (unsigned integer) |

**매뉴얼 Section 1.5**: 카드 번호는 unsigned integer. `1`과 `001`은 동일.
**판정**: 기능 차이 없음.

### 2.5 Card 115 (Noncondensable Mass Fractions)

| 카드 | ICV | PRI |
|------|-----|-----|
| 115 | (없음) | `115  1.0000` |

**매뉴얼 Section 2.13**: Card 115는 비응축성 기체 질량 분율. Card 110에 `air` 하나만 지정된 상태에서 기본값은 1.0.
**판정**: 기능 차이 없음 (PRI가 기본값을 명시적으로 입력).

### 2.6 제어변수 번호 포맷

| 카드 | ICV | PRI |
|------|-----|-----|
| 20501501 W2 | `cntrlvar 014` | `cntrlvar 14` |

**판정**: 기능 차이 없음 (leading zero 무시).

### 2.7 Minor Edit Card 301 중복

| 카드 | ICV | PRI |
|------|-----|-----|
| 301 (1차) | `301  rktpow  0` | - |
| 301 (2차) | `301  cntrlvar  325` | `301  cntrlvar  325` |

**매뉴얼 Section 1.2**: 중복 카드 번호는 마지막 것만 사용.
ICV에서 `301 rktpow 0`은 뒤의 `301 cntrlvar 325`로 덮어씌워짐.
**판정**: 기능 차이 없음.

### 2.8 SG 튜브 열구조물 경계조건 (HS 1310/1330/1350/1370)

ICV에서 중복 카드 패턴 사용:
```
13100501  310010000  0  101  1  1552.5  1  ← 첫 번째 (convective)
13100501  0          0  0    1  1552.5  1  ← 두 번째 (insulated) ← 마지막, 유효
```

MARS "마지막 카드 우선" 규칙에 의해 ICV 유효값은 **insulated (W1=0)**.
PRI도 `0 0 0 1 1552.5 1` (insulated) 출력.

**판정**: 기능 차이 없음. PRI가 정확하게 출력함. 파서의 `deduplicateCards()` 정상 동작.

## 3. GUI 입력 차이 (파서 버그 아님, 데이터 소스 차이)

아래 2건은 파서/생성기 버그가 아니라, GUI에서 사용자가 다른 값을 입력했거나 다른 데이터 소스에서 가져온 것으로 판단됨.

### 3.1 Card 1150101-1150102: Core Bypass 입구 정션 (inlet_cb) 손실계수

**컴포넌트**: C115 (inlet_cb, SNGLJUN) - Core Bypass 입구 정션

**ICV (원본)**:
```
1150101  110010002  140010001  0.141145        ← W1-W3 (from, to, area)
1150102  10.0       10.0       00000000        ← 첫 번째 1150102
1150102  07.0       07.0       00000000        ← 두 번째 1150102 (마지막 = 유효)
```
마지막 `1150102` 적용: **fwd_loss = 7.0, rev_loss = 7.0**, flags = 00000000

**PRI (GUI 생성)**:
```
1150101  110010002  140010001  0.141145  76.0  7.0  00000000
```
- **fwd_loss = 76.0, rev_loss = 7.0**, flags = 00000000

| 항목 | ICV (유효값) | PRI | 차이 |
|------|------------|-----|------|
| Forward Loss (W4) | **7.0** | **76.0** | 값이 다름 |
| Reverse Loss (W5) | **7.0** | **7.0** | 동일 |

파서는 ICV의 중복 카드를 올바르게 처리하여 7.0/7.0을 읽음. **76.0은 ICV 파일에 존재하지 않는 값**으로, GUI에서 수동 변경되었거나 다른 데이터 소스에서 유래한 것.

**영향**: Core Bypass 순방향 유동 저항이 ICV 대비 10.9배 높음.

---

### 3.2 Card 11100501: Heat Structure 1110 좌측 경계조건 볼륨

**컴포넌트**: HS 1110 (core_p에 대한 열구조물) 좌측 경계

ICV에 중복 카드 없음. 값 자체가 다름.

| 항목 | ICV | PRI |
|------|-----|-----|
| W1 (Boundary Volume) | **110010000** (core_p, vol 01) | **260010000** (bt_head, vol 01) |
| W3 (Boundary Type) | 101 (convective) | 101 (convective) |

**매뉴얼 Section 9.16**: W1은 열구조물 좌측 표면과 연결되는 수력학적 볼륨 번호.

이 차이는 중복 카드 문제가 아니며, GUI에서 다른 값이 입력된 것.

**영향**: 열구조물의 좌측 경계에서 대류 열전달을 계산할 때 참조하는 유체 볼륨이 다름.
- ICV: 코어 지지판(core_p) 볼륨 110 온도 참조
- PRI: 하부 헤드(bt_head) 볼륨 260 온도 참조

## 4. 근본 원인 분석

**파서/생성기는 정상 동작**: `grouper.ts`의 `deduplicateCards()` 함수가 MARS "마지막 카드 우선" 규칙을 올바르게 구현하고 있으며, 중복 카드 처리가 정확함.

2건의 실질적 차이는 모두 **GUI에서의 데이터 입력 차이**로 판단:

1. **C115 fwd_loss = 76.0**: ICV에 없는 값. GUI에서 사용자가 수동 변경했거나 다른 데이터 소스에서 로드한 것으로 추정.
2. **HS 1110 boundary = 260010000**: ICV에 중복 카드 없이 단일 값(110010000)이며, GUI에서 다른 값으로 변경된 것.

## 5. 결론

### 파서/생성기 검증 결과
- **형식적 차이** (숫자 표기, 주석, leading zero, 중복 카드): 약 2,000건 이상 → 모두 **정확하게 처리됨**
- 중복 카드 패턴 (SG 튜브 HS, Minor Edit 301 등): **올바르게 마지막 카드 적용**
- **파서 및 파일 생성기에 버그 없음**

### 데이터 차이 2건
| 위치 | ICV (유효값) | PRI | 원인 |
|------|------------|-----|------|
| C115 fwd_loss (W4) | 7.0 | 76.0 | GUI 입력 차이 |
| HS 1110 boundary volume (W1) | 110010000 | 260010000 | GUI 입력 차이 |

이 2건은 ICV.i 파일에서 그대로 파싱된 결과가 아니라, GUI에서 별도로 입력/변경된 값임.
ICV.i와 동일한 결과를 원한다면 GUI에서 해당 2개 값을 수정해야 함.
