# MARS 입력 파일 컴포넌트 비교 분석

## 개요

**비교 대상:**
- 파일1: `SMART_SIM_BASE_REV01.i` (기존 시뮬레이터 모델, 11,090줄)
- 파일2: `SMART(100-50-100).i` (BOP 포함 모델, 14,732줄)

**분석 목적:** 파일2 지원을 위한 추가 구현 범위 파악 및 기존 파일 호환성 평가

---

## 1. 컴포넌트 타입별 수량 비교

| 타입 | 매뉴얼 섹션 | 파일1 (BASE) | 파일2 (BOP) | 차이 | 비고 |
|------|-----------|:-----------:|:---------:|:---:|------|
| pipe | 8.6 | 49 | 95 | +46 | BOP 배관 대량 추가 |

### 핵심 변화

- **신규 3종** (turbine, tank, separatr): 파일1에 없던 타입으로, 추가 구현 필요
- **대량 증가**: pipe(+46), sngljun(+34) — BOP 계통 모델링으로 인한 증가
- **감소 타입**: tmdpvol(-11), valve(-10), branch(-8) — BOP를 경계조건이 아닌 실제 컴포넌트로 모델링하여 경계조건 수 감소

## 2. 매뉴얼 대비 불일치 분석 (전체 12종)

### 2.1 매뉴얼 완전 일치 (9종) — 추가 구현 불필요

| 타입 | 인스턴스 | 비고 |
|------|:-------:|------|
| pipe | 95 | word 수 100% 일치. 카드 중복 ~7건 (기능 무해) |
| sngljun | 63 | 매뉴얼 100% 준수. 카드 중복 4~6건 |
| branch | 3 | 불일치 0건, 완벽 일치 |
| tmdpjun | 3 | 불일치 0건, 완벽 일치 |
| pump | 6 | 형식 불일치 0건. Homologous 커브 16개 검증 완료 |
| mtpljun | 6 | NNM 체계 정상. 주석 순서 경미 1건 |
| snglvol | 4 | 카드 중복 1건 (낮음) |
| tmdpvol | 6 | C292 볼륨 플래그 위반 1건 (중간) |
| valve | 18 | 중복 선언 7건 (trip valve↔motor valve 전환 흔적) |

### 2.2 매뉴얼 불일치 — 추가 구현 필요 (3종)

#### turbine (8개) — 구현 난이도: 중

| 카드 | 발견 사항 | 심각도 |
|------|-----------|:------:|
| CCC0181 | Y좌표 볼륨 데이터 — 매뉴얼 8.13에 미정의 (Branch 8.10에만 존재) | 높음 |
| CCC0300 | 초기조건 중복 정의 (shaft comp 004→999 변경) | 중간 |
| 6102101 주석 | "Homogeneous ON" 주석이나 실제 플래그는 choking만 | 낮음 |

- Branch 기반 확장으로 대응 가능
- CCC0181 카드 처리 방침 결정 필요

#### separatr (1개) — 구현 난이도: 중

| 카드 | 발견 사항 | 심각도 |
|------|-----------|:------:|
| CCC0102 W9 | 볼륨 플래그 f=0 (wall friction ON) — 매뉴얼은 f=1 (OFF) 요구 | 높음 |
| CCC1101 W1 | N=1 junction from: face 0 사용 — 매뉴얼은 face 2 요구 | 높음 |
| CCC2101 W1 | N=2 junction from: vol 00 face 0 — 매뉴얼은 vol 01 face 1 요구 | 높음 |

- 매뉴얼 vs 실제 입력 불일치가 의도적 모델링인지 입력 오류인지 판단 필요

#### tank (6개) — 구현 난이도: 높음

| 카드 | 발견 사항 | 심각도 |
|------|-----------|:------:|
| CCC0000 | 키워드 `tank` — 매뉴얼은 `accum`만 정의 | 높음 |
| CCC0001 | junction 개수 카드 (값=2) — ACCUM 매뉴얼에 없음 | 높음 |
| CCC0200 | 플래그 기반 초기조건 (SNGLVOL 유사) — ACCUM은 직접 P,T,boron | 높음 |
| CCCN101 | from+to 코드 7 words (SNGLJUN 유사) — ACCUM은 to만 5 words | 높음 |
| CCCN201 | junction 초기조건 카드 존재 — ACCUM에 없음 | 높음 |
| — | 2-junction 지원 — ACCUM은 1-junction만 | 높음 |
| W5 | inclination=0 허용 — ACCUM은 ±90도만 | 중간 |

- ACCUM과 **완전히 다른 컴포넌트 타입** — 별도 신규 구현 필요

## 3. 파일 간 공통 패턴 차이

| 항목 | 파일1 (BASE) | 파일2 (BOP) |
|------|-------------|------------|
| 모델 범위 | NSSS + 2차측 경계조건 | NSSS + BOP 전체 (터빈~급수가열기) |
| 출력 | 330 MW | 365 MW |
| 운전 시나리오 | 정상상태 | 부하추종 (100%→50%→100%) |
| 카드 분할 방식 | 3+3+3 word 분할 | NSSS: 3+3+3 / BOP: 6~9 word 한 줄 |
| 연결 코드 포맷 | 확장 포맷 (cccvv000n) | NSSS: 확장 / BOP: 구 포맷 혼용 |
| 중복 카드 (override) | 거의 없음 | 약 25건 (값 수정 흔적) |
| 제어변수/shaft | 없음 | CV999 (shaft/generatr 정의) |

## 4. 결론

### 기존 프로젝트 파일 재사용: **불가**

1. 신규 컴포넌트 3종(turbine, tank, separatr)이 현재 미지원
2. BOP 영역 컴포넌트 63개 추가 (pipe +46, sngljun +34 등)
3. 카드 분할 방식, 연결 코드 포맷이 영역별로 상이
4. 중복 카드를 통한 값 override가 다수 존재하여 단순 변환 불가

### 추가 구현 항목

| 우선순위 | 항목 | 비고 |
|:--------:|------|------|
| 1 | tank 컴포넌트 타입 신규 정의 | ACCUM과 별도 설계 필요 |
| 2 | turbine 컴포넌트 구현 | Branch 기반 확장 + CCC0181 처리 |
| 3 | separatr 컴포넌트 구현 | 매뉴얼 불일치 해석 후 결정 |
| 4 | 입력 파일 파서의 중복 카드 처리 | last-wins 규칙 적용 |
| 5 | BOP 영역 연결 코드 포맷 호환 | 구 포맷/확장 포맷 혼용 대응 |
